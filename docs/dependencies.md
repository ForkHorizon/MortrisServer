# Dependency and license inventory (section 14, Phase S4)

Generated 2026-07-17 from `go list -deps ./cmd/analytics-server` (the
actual compiled package graph, not `go list -m all`'s full module graph —
several test-only transitive modules like `testify`/`go-spew` show up in
the latter but never get linked into the binary) and `npm ls
--all --depth=0` in `dashboard/`.

All licenses below are permissive (MIT, BSD-3-Clause, Apache-2.0) — no
copyleft (GPL/AGPL) dependencies anywhere in the tree, so there's nothing
here that constrains keeping this a closed-source internal tool.

## Known gaps

- `oxlint` is present but not run in `make lint` or CI — only Go's own
  `go vet`/`gofmt` are wired in. Add an `npm run lint` step if frontend
  lint coverage becomes a priority.
- This inventory is generated by hand, not by an automated tool
  (`go-licenses`, `license-checker`, etc.) — re-run the two commands
  above and re-diff against this file before a license question actually
  matters (an audit, a customer contract, distributing the binary
  externally), rather than trusting this document's staleness blindly.
